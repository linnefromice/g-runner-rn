# Auto Merge PR

ローカルの変更からブランチ作成 → コミット → プッシュ → PR作成 → CI待機 → マージ → デフォルトブランチに戻るまでを一括で実行する。

## 引数

`$ARGUMENTS` が指定されていれば、PRのタイトルや説明のヒントとして使用する。

## 手順

### Phase 1: PR 作成

`/create-pr` の手順をそのまま実行する。`$ARGUMENTS` があれば渡す。

PR が作成されたら、その PR 番号を記録して Phase 2 に進む。

### Phase 2: CI 確認 & マージ

Phase 1 で作成された PR に対して `/merge-pr` の手順を実行する。

## CI 失敗時のフロー

CI が失敗した場合は Phase 2 で停止する。ユーザーに以下を報告:

1. 失敗した check の名前と詳細
2. PR の URL（手動で確認可能）
3. 「修正後に `/merge-pr` で手動マージできます」と案内

修正を自動で試みたりはしない。

## ルール

- `/create-pr` と `/merge-pr` のルールをすべて継承する
- CI が失敗している場合は絶対にマージしない
- 修正の自動試行は行わない — 報告して停止
