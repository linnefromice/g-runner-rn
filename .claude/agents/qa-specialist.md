## テスト戦略

### テストピラミッド

```
        ┌─────────┐
        │  E2E    │  少数・高コスト（Detox/Maestro）
        ├─────────┤
        │  統合   │  中程度（システム間連携）
        ├─────────┤
        │ユニット │  多数・低コスト（Jest）
        └─────────┘
```

### テスト種別と対象

| テスト種別   | 対象                                 | ツール        |
| ------------ | ------------------------------------ | ------------- |
| ユニット     | エンジンシステム、ユーティリティ     | Jest          |
| 統合         | システム間連携、ストア更新           | Jest          |
| パフォーマンス | フレームレート、メモリ使用量       | Perf Monitor  |
| E2E          | ゲームフロー全体                     | Detox/Maestro |

## テストケース設計

### ゲームエンジンテスト

| テストケース       | 前提条件           | 操作               | 期待結果               |
| ------------------ | ------------------ | ------------------ | ---------------------- |
| 衝突判定（AABB）   | 2エンティティ重なり | collisionSystem実行 | 衝突検出               |
| 衝突判定（非接触） | 2エンティティ離間   | collisionSystem実行 | 衝突なし               |
| 移動システム       | velocity設定済み    | movementSystem実行  | position更新           |
| スコア計算         | コンボ状態あり      | addScore呼び出し    | コンボ倍率適用         |
| HP管理             | HP 3/5             | takeDamage(2)       | HP 1/5, i-frame開始   |
| コンボリセット     | コンボ2段目        | トレードオフゲート通過 | コンボリセット       |

### ゲートシステムテスト

| テストケース     | ゲートタイプ | 操作           | 期待結果                    |
| ---------------- | ------------ | -------------- | --------------------------- |
| 強化ゲート通過   | enhance      | プレイヤー通過 | ステータス強化              |
| リフィットゲート | refit        | プレイヤー通過 | フォーム変更、コンボリセット |
| トレードオフ     | tradeoff     | プレイヤー通過 | ステータスA↑B↓              |
| 回復ゲート       | recovery     | プレイヤー通過 | HP回復                      |

### ボスフェーズテスト

| テストケース       | 前提条件       | 操作             | 期待結果                   |
| ------------------ | -------------- | ---------------- | -------------------------- |
| ボスフェーズ開始   | タイムライン到達 | フェーズ遷移     | 背景0.5x、スポーン停止     |
| ボス撃破           | ボスHP 0       | ダメージ適用     | ステージクリア             |
| ボスタイムアウト   | 制限時間経過   | タイマー満了     | ゲームオーバー or 次フェーズ |

## エッジケース・異常系

### 座標境界

| シナリオ           | 入力             | 期待動作               |
| ------------------ | ---------------- | ---------------------- |
| 画面外（左）       | x < 0            | x = 0 にクランプ       |
| 画面外（右）       | x > 320          | x = 320 にクランプ     |
| 画面外（上）       | y < 0            | エンティティ削除       |
| 画面外（下）       | y > screenHeight | エンティティ削除       |

### エンティティ数

| シナリオ             | 期待動作                       |
| -------------------- | ------------------------------ |
| エンティティ 50+     | 60fps維持                      |
| エンティティ 100+    | フレーム落ちの程度を計測       |
| 弾幕 + エフェクト重畳 | メモリ使用量が閾値以下        |

### 状態遷移

| 現在状態       | 操作               | 期待動作           |
| -------------- | ------------------ | ------------------ |
| i-frame中      | 被弾               | ダメージ無効       |
| Awakened form中 | ダメージ           | Awaken解除判定     |
| ポーズ中       | ゲームシステム更新 | 更新停止           |
| ゲームオーバー | タッチ入力         | リザルト画面表示   |

## パフォーマンステスト

### ターゲットデバイス

| デバイス           | 基準                  |
| ------------------ | --------------------- |
| iPhone SE 2nd gen  | 60fps安定（最低ライン）|
| iPhone 13          | 60fps安定             |
| Pixel 6            | 60fps安定             |

### パフォーマンス基準

| 指標                     | 目標         | 説明                           |
| ------------------------ | ------------ | ------------------------------ |
| フレームレート           | 60fps 安定   | 50fps 以下は NG                |
| JS スレッド更新予算      | ≤ 16ms       | 全システム合計                 |
| タッチ入力レイテンシ     | ≤ 33ms       | 2フレーム以内                  |
| メモリ使用量             | < 200MB      | 長時間プレイでリーク無し       |
| エンティティ同時数       | 50+          | 弾 + 敵 + エフェクト          |

### テスト項目

- [ ] 通常ステージでの60fps維持
- [ ] ボスフェーズでの60fps維持
- [ ] 弾幕時（50+エンティティ）のフレームレート
- [ ] 5分間連続プレイ後のメモリ使用量
- [ ] ステージ間遷移時のローディング時間
