Phase 2-A（基礎修正）と Phase 2-B（コンテンツ拡充）が完了していること。

## 1. グレイズシステム (Graze)

### コンセプト
敵弾がプレイヤーの見た目（32×40px）には重なるが、ヒットボックス（16×16px）には当たらない「カスり」状態を検知。
カスりに報酬を与えることで、上級者が意図的に弾幕スレスレを飛ぶリスク＆リターンの遊びを生む。

### 判定方法
```
敵弾の位置が以下の範囲に入ったらグレイズ:
  プレイヤー visual hitbox (32×40) と重なる
  AND プレイヤー actual hitbox (16×16) とは重ならない
```

つまり visual hitbox と actual hitbox の間のリング状領域がグレイズゾーン。

### 報酬
| 項目 | 値 | 備考 |
|------|-----|------|
| EX ゲージ | +3/弾 | 通常の敵撃破(+5)の60% |
| TF ゲージ | +2/弾 | 通常の敵撃破(+8)の25% |
| スコア | +20/弾 | 少量だが積み重なる |
| 効果音 | 「ｼﾞｼﾞｯ」 | Phase 3 で実装 |
| エフェクト | 火花パーティクル | Phase 3 で実装 |

### 制約
- 同一弾に対するグレイズは1回のみ（弾ごとにフラグ管理）
- Awakened 中は無敵なのでグレイズ判定なし（すべてヒットボックス内に入れる）

### 変更対象ファイル
- `src/types/entities.ts` — `BulletEntity` に `grazed?: boolean` 追加
- `src/engine/systems/CollisionSystem.ts` — グレイズ判定追加（既存の弾→プレイヤー判定の前に）
- `src/constants/balance.ts` — グレイズ報酬定数
- `src/constants/dimensions.ts` — `PLAYER_VISUAL_HITBOX` 定数（既に PLAYER_VISUAL として存在）

## 2. ジャスト TF（パリィ）

### コンセプト
敵の攻撃がヒットする直前（猶予フレーム内）に TF ボタンを押して形態変化すると「ジャスト TF」が発動。
被ダメージを無効化し、周囲に衝撃波を放つ。ロボットアニメの「変形回避」を再現。

### 判定方法
```
TF ボタン押下時に以下をチェック:
  1. Transform ゲージが MAX
  2. 直近の N フレーム以内に被弾判定が発生する予定
     → 実装上は「TF 発動後の猶予時間内に被弾があった場合に無効化」が現実的
```

### 設計パラメータ（仮）
| パラメータ | 値 | 備考 |
|-----------|-----|------|
| パリィ猶予時間 | 200ms (12フレーム) | TF 押下後の無敵猶予 |
| 衝撃波半径 | 60px | プレイヤー中心から |
| 衝撃波ダメージ | 30 | 周囲の敵にダメージ |
| 弾消し | 衝撃波範囲内の敵弾を全消去 | |
| ボーナススコア | +300 | ジャスト TF 成功時 |
| EX ゲージ | +15 | 大きめの報酬 |
| 演出 | 衝撃波リング + スローモーション(100ms) | Phase 3 で実装 |

### 実装方針
- TF ボタン押下時に `justTFTimer = 200ms` を設定
- `justTFTimer > 0` の間に被弾が発生した場合:
  - ダメージを0に
  - 衝撃波発動（周囲の敵にダメージ + 敵弾消去）
  - ボーナス報酬
- `justTFTimer` が0になった場合は通常の TF として処理

### 変更対象ファイル
- `src/stores/gameSessionStore.ts` — `justTFTimer` 状態追加
- `src/engine/systems/CollisionSystem.ts` — パリィ判定追加
- `src/engine/systems/TransformGaugeSystem.ts` — ジャスト TF タイマー管理
- `src/constants/balance.ts` — パリィ関連定数

## 3. グロウゲート (Growth Gate)

### コンセプト
プレイヤーの弾を当てるとゲート効果値が上昇する特殊ゲート。
「より強い効果を得るためにゲート前に留まる」vs「敵の攻撃を避ける」のジレンマを生む。

### 仕様
- ゲートのタイプマーカー: `'growth'`（Enhance のサブタイプとして扱う）
- 初期値: 例 `ATK +3`
- 弾1発当たるごとに `+1` 増加（上限あり）
- 通過時の値で効果確定

### 設計パラメータ（仮）
| パラメータ | 値 | 備考 |
|-----------|-----|------|
| 初期効果値 | ベース効果の50% | 例: ATK+3（通常のATK+5の60%） |
| 成長量 | +1/ヒット | |
| 上限 | ベース効果の300% | 例: ATK+15 |
| 弾の消滅 | ゲートに当たった弾は消える | 敵への弾が減るトレードオフ |
| 表示 | ラベルがリアルタイムで更新 | `ATK +3` → `ATK +4` → ... |

### 変更対象ファイル
- `src/types/gates.ts` — `GateEntity` に `growthHits?: number`, `maxGrowth?: number` 追加
- `src/engine/systems/CollisionSystem.ts` — 弾→ゲート の衝突判定追加
- `src/engine/systems/GateSystem.ts` — 通過時に成長後の値で効果適用
- `src/game/gates.ts` — Growth gate 定義追加

## 4. ルーレットゲート (Roulette Gate)

### コンセプト
ゲートの表示効果が一定間隔で切り替わるタイミングゲート。
良い効果と悪い効果が交互に表示され、通過タイミングで結果が決まる。

### 仕様
- 表示が 0.5 秒ごとに2つの効果を交互に切り替え
- 通過した瞬間の表示効果が適用される
- 例: `[ATK +10]` ↔ `[HP -20]`

### 設計パラメータ（仮）
| パラメータ | 値 | 備考 |
|-----------|-----|------|
| 切替間隔 | 500ms | 0.5秒ごと |
| 効果パターン | 良い効果 ↔ 悪い効果 | プラスとマイナスの交互 |
| コンボ影響 | Enhance として +1 | 良い効果で通過した場合 |
| 視覚 | ゲートの色が切替に合わせて変化 | Phase 3 で実装 |

### バリエーション例
| ペア名 | 効果A (良) | 効果B (悪) |
|--------|-----------|-----------|
| ROULETTE_ATK | ATK +10 | ATK -5 |
| ROULETTE_HP | HP +50 | HP -30 |
| ROULETTE_EXTREME | ATK +20 | SPD ×0.5 |

### 変更対象ファイル
- `src/types/gates.ts` — `GateEntity` に `rouletteEffects?: GateEffect[][]`, `rouletteTimer?: number` 追加
- `src/engine/systems/GateSystem.ts` — ルーレット状態の更新と通過時の効果決定
- `src/game/gates.ts` — ルーレットゲート定義

## 5. デブリ（障害物）

### コンセプト
弾を通さず、接触するとダメージを受ける環境オブジェクト。
形態の特殊能力（爆発で破壊、貫通で無視）を活かす場面を作る。

### 仕様
- 新エンティティタイプ: `'debris'`
- スクロールで上から降りてくる（敵と同じ速度）
- プレイヤー弾を吸収（弾が消える）
- プレイヤーが接触するとダメージ
- 破壊不可（ただし爆発弾は例外）

### 設計パラメータ（仮）
| パラメータ | 値 | 備考 |
|-----------|-----|------|
| ヒットボックス | 40×40px | 敵より大きい |
| 接触ダメージ | 20 | |
| HP | 50 | 爆発弾でのみ破壊可能（通常弾は吸収） |
| 弾吸収 | 通常弾・ホーミング弾を消す | |
| 爆発弾 | ダメージを与える（破壊可能） | |
| 貫通弾 | デブリを貫通して通過 | 弾もデブリも消えない |
| スコア | 50 (破壊時) | |

### 形態との関係
- Standard: デブリを避けるしかない
- **Heavy Artillery: 爆発弾でデブリ自体を破壊**
- **High Speed: 貫通弾がデブリを通り抜けて後ろの敵に当たる**
- Awakened: ホーミング弾はデブリに吸われる → デブリ破壊不可

### 変更対象ファイル
- `src/types/entities.ts` — `DebrisEntity` 型追加
- `src/engine/entities/Debris.ts` — 新規: デブリ生成
- `src/engine/systems/CollisionSystem.ts` — 弾→デブリ / プレイヤー→デブリ 判定
- `src/types/stages.ts` — `StageEvent` に `'debris_spawn'` 追加
- `src/engine/systems/SpawnSystem.ts` — デブリスポーン対応

## 6. 加速レーン (Boost Lane)

### コンセプト
画面の一部に存在する光る帯。乗っている間はスクロール速度が上がり、スコア倍率がアップ。
ハイスコアを狙うなら加速レーンに乗り続けるが、敵の対処が難しくなるリスク。

### 仕様
- 画面の特定X座標範囲に出現（例: 中央80px幅）
- レーン上にいる間:
  - スコア倍率 ×1.5
  - スクロール速度 ×1.3（敵・ゲートが速く降りてくる）
- レーンから外れると通常に戻る

### 設計パラメータ（仮）
| パラメータ | 値 | 備考 |
|-----------|-----|------|
| レーン幅 | 80px (論理座標) | 画面幅の25% |
| 出現位置 | ステージごとに異なるX座標 | タイムラインで制御 |
| スコア倍率 | ×1.5 | レーン上にいる間のみ |
| スクロール倍率 | ×1.3 | 敵もゲートも速くなる |
| 持続時間 | 10〜20秒 | タイムラインイベントで on/off |

### 変更対象ファイル
- `src/types/stages.ts` — `StageEvent` に `'boost_lane'` 追加
- `src/engine/systems/ScrollSystem.ts` — レーン判定 + スクロール倍率
- `src/stores/gameSessionStore.ts` — `scoreMultiplier` 追加
- `src/game/scoring.ts` — 倍率適用

## 実装順序

```
1. グレイズシステム
   ← CollisionSystem の拡張で済む、他への依存なし
2. デブリ（障害物）
   ← 新エンティティだが、爆発弾・貫通弾の仕組みを活用
3. グロウゲート
   ← ゲートシステムの拡張
4. ルーレットゲート
   ← ゲートシステムの拡張
5. 加速レーン
   ← ScrollSystem + スコアシステムの拡張
6. ジャスト TF
   ← TransformGaugeSystem + CollisionSystem の連携（最も複雑）
```
